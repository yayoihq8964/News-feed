## 1. 产品概述

**MacroLens** 是一个面向宏观投资者的 AI 驱动新闻情绪分析平台。核心价值主张是：

> 聚合多源金融新闻 → AI 情绪分析 → 可视化市场脉搏 → 辅助投资决策

**目标用户画像**：
- 关注美股与贵金属市场的散户/半专业投资者
- 需要快速获取宏观新闻情绪判断的量化交易员
- 希望用中文阅读英文金融新闻的华语投资者

**技术栈**：React 18 + TypeScript + Tailwind CSS（前端）| Python FastAPI + SQLite + 多 LLM 提供商（后端）

## 3. 用户体验评估

### 3.1 信息架构 (IA) — ⭐⭐⭐⭐ (4/5)

**优点**：
- 四页架构清晰：Markets → News → Sentiment → Analysis
- 遵循「概览→列表→详情」的渐进式信息披露模式
- 侧边栏复用经济日历/情绪指数/大宗商品，信息密度合理

**不足**：
- Markets 和 Sentiment Dashboard 有大量重复内容（指数卡片、商品卡片、Fear&Greed），用户可能困惑两者差异
- Deep Analysis 页面缺少从列表到详情的平滑过渡（无面包屑导航动画）

### 3.2 视觉设计 — ⭐⭐⭐⭐⭐ (5/5)

**优点**：
- Material Design 3 色彩系统 + Violet 主色调，识别度高
- Manrope + Inter 字体组合专业感强
- Glass effects（backdrop-blur）层次分明
- Dark mode 实现完善，色彩对比度达标
- 情绪标签的语义化配色（绿=多/红=空/灰=中性）直觉性强

**不足**：
- 迷你柱状图 `Math.random()` 生成随机数据，视觉上误导用户认为是真实趋势

### 3.3 交互设计 — ⭐⭐⭐ (3/5)

**优点**：
- 操作按钮（抓取新闻/AI 分析）位置显眼，主次分明
- Polling 自动刷新，用户无需手动刷新页面
- 新闻卡片 hover 效果和过渡动画流畅

**不足**：
- **无操作反馈**：点击"AI分析"后仅靠 `setTimeout` 延迟刷新，无进度条/skeleton 加载
- **无确认对话框**：触发分析等操作无成本确认（会消耗 LLM token）
- **空状态设计不完善**：部分场景仅显示"加载中..."，缺少空状态插画引导
- **无手势支持**：移动端缺少左滑删除/下拉刷新等原生交互
- **侧边栏在 xl 以下完全隐藏**：平板用户丢失经济日历/情绪指数信息，无折叠入口

### 3.4 可用性问题清单

| 问题 | 严重度 | 位置 | 描述 |
|------|--------|------|------|
| 假数据柱状图 | 🔴 高 | Markets / Sentiment 指数卡片 | `Math.random()` 生成的柱状图看似价格趋势但无真实数据支撑，严重误导 |
| 中英混杂 | 🟡 中 | Deep Analysis 页面 | "Run Analysis"/"Back to News Feed" 等按钮未翻译，与中文界面不一致 |
| 无分析进度反馈 | 🟡 中 | NewsFeed "AI分析" 按钮 | 点击后 3 秒 setTimeout 刷新，用户不知道是否成功 |
| API Key 安全 | 🔴 高 | Settings 页 | 无认证保护，任何访问者可查看/修改 API Key |
| 侧边栏断层 | 🟡 中 | < xl 屏幕 | 经济日历、情绪指数在中等屏幕完全消失 |
| 新闻无限加载 | 🟢 低 | NewsFeed | 固定 25 条无分页，历史新闻不可访问 |

## 4. 产品定位与竞品差异化

### 4.1 竞品对比矩阵

| 维度 | MacroLens | MacroMicro | TradingView 新闻 | Bloomberg Terminal |
|------|-----------|------------|-------------------|-------------------|
| **目标用户** | 散户/半专业 | 宏观研究员 | 全类型交易者 | 机构/专业 |
| **核心价值** | AI 情绪分析 | 宏观指标可视化 | 社区+技术分析 | 全方位终端 |
| **新闻聚合** | ✅ 6 源聚合 | ❌ 无独立新闻流 | ✅ 内置新闻流 | ✅ 专业新闻社 |
| **AI 分析** | ✅ 核心差异点 | ❌ | ❌ 无 AI 分析 | 🟡 有限 AI |
| **中文支持** | ✅ AI 翻译 | ✅ 原生中文 | 🟡 部分中文 | ❌ 英文为主 |
| **行情数据** | 🟡 延迟行情 | ✅ 丰富指标 | ✅ 实时行情 | ✅ 实时+深度 |
| **经济日历** | ✅ AI 增强 | ✅ 深度日历 | ✅ 基础日历 | ✅ 专业日历 |
| **社交情绪** | 🟡 LLM 估算 | ❌ | ✅ 社区情绪 | ✅ 真实数据 |
| **价格** | 免费/自部署 | 付费订阅 | 免费+付费 | $24,000/年 |
| **自部署** | ✅ Docker | ❌ SaaS | ❌ SaaS | ❌ 专用终端 |

### 4.2 MacroLens 的差异化优势

1. **AI-Native 情绪分析**：这是核心差异点。竞品中没有任何产品提供"新闻→结构化情绪分析→受影响资产拆解→逻辑链推理"的端到端 AI 分析链路
2. **自部署 + 开源**：隐私敏感的投资者可完全自托管，API Key 本地管理
3. **多 LLM 灵活切换**：支持 OpenAI/Claude/Grok/Ollama 本地模型，避免供应商锁定
4. **经济日历 AI 增强**：对日历事件做多空判断，这在主流产品中罕见
5. **中文翻译层**：解决华语投资者阅读英文宏观新闻的痛点

### 4.3 当前劣势 & 需弥补的差距

1. **行情数据深度不足**：仅 7 个品种，无 K 线图、无技术指标，与 TradingView 差距巨大
2. **社交情绪为 LLM 估算而非真实数据**：与 Bloomberg/TradingView 的真实社交数据相比说服力不足
3. **无社区/协作功能**：TradingView 的核心壁垒是社区，MacroLens 完全没有
4. **数据历史深度有限**：SQLite 单机存储，无法与 MacroMicro 的长周期宏观数据库竞争

### 4.4 推荐定位

> **"面向华语宏观投资者的 AI 情绪分析个人终端"**

- 不应与 TradingView 竞争行情图表（差距太大且非核心价值）
- 不应与 MacroMicro 竞争宏观数据深度（数据获取壁垒高）
- 应强化 **AI 分析深度** + **个人工作流集成** + **自部署隐私** 三个维度

## 6. MVP 评估

### 6.1 MVP 核心假设验证

| 假设 | 是否已验证 | 评估 |
|------|-----------|------|
| 用户需要 AI 分析宏观新闻的情绪 | ✅ 功能可用 | 核心分析链路完整，可产出结构化分析 |
| AI 分析结果对投资决策有参考价值 | 🟡 需验证 | 分析质量依赖 LLM 模型，缺少准确率回测 |
| 用户需要中文翻译英文新闻 | ✅ 功能可用 | 翻译 + 摘要质量取决于 LLM |
| 多源聚合比单一来源更有价值 | ✅ 功能可用 | 6 源聚合 + 去重机制完整 |
| 用户需要经济日历的 AI 增强 | ✅ 功能可用 | 已实现日历事件的多空分析 |

### 6.2 MVP 达标评估

**结论：✅ 基本达到 MVP 标准，但有两个关键风险需立即修复**

**达标理由**：
1. 端到端的核心价值链路（聚合→分析→可视化）完整闭合
2. 4 个主要页面（Markets/News/Sentiment/Analysis）均可使用
3. 后端调度系统自动化运行，无需手动干预
4. 多 LLM 提供商支持，降低了单点依赖风险
5. Docker 部署方案降低了使用门槛

**未达标风险**：
1. 🚨 **假数据问题**：柱状图 `Math.random()` 可能导致用户误判市场趋势，对于金融产品这是不可接受的
2. 🚨 **安全问题**：无认证保护下暴露 API Key 和 LLM 操作，部署在公网即被滥用
3. ⚠️ **无测试覆盖**：金融数据分析产品没有测试，迭代时回归风险极高

## 7. 产品迭代路线图

### Phase 1：MVP 修复与稳固（第 1-2 周）

**目标**：消除关键风险，使产品可安全使用

```
Week 1:
├── [P0] 移除 Math.random() 柱状图，改为真实历史数据 mini sparkline 或隐藏
├── [P0] 添加基础认证（HTTP Basic Auth 或简单 token）
├── [P0] 后端添加 API Rate Limiting（slowapi/ratelimit）
├── [P0] 统一所有 UI 文案为中文
└── [P1] 添加 React Error Boundary

Week 2:
├── [P1] 实现 Toast 通知系统（操作成功/失败反馈）
├── [P1] 新闻列表分页（前端分页控件 + 后端已支持）
├── [P1] 实现收藏/置顶功能（is_pinned 字段前端接通）
```

## 8. 总结与风险提示

### 8.1 核心结论

MacroLens 作为一个 AI 驱动的宏观新闻情绪分析平台，**核心价值链路（新闻聚合 → AI 分析 → 情绪可视化）已完整实现**，在金融 AI 工具市场中具有差异化定位。视觉设计水准高于同类开源项目，后端架构（多 LLM 提供商、定时调度、去重机制）设计合理。

但产品存在 **两个需立即修复的关键问题**：假数据柱状图和安全认证缺失。这两个问题在金融产品语境下风险极高，应在任何新功能开发之前优先解决。

### 8.2 关键风险

| 风险 | 级别 | 缓解措施 |
|------|------|---------|
| `Math.random()` 柱状图误导用户 | 🔴 致命 | Phase 1 第一天移除或替换 |
| 无认证导致 API Key 泄露 | 🔴 致命 | Phase 1 添加 Basic Auth |
| LLM token 成本失控 | 🟡 高 | Rate Limiting + 批量分析上限 |
| 社交情绪为 LLM 估算非真实数据 | 🟡 高 | 明确标注"AI 估算"（已做） + Phase 3 接入真实数据 |
| SQLite 单机存储性能瓶颈 | 🟢 中 | 当前规模可用，Phase 3 迁移 PostgreSQL |
| 无测试覆盖导致迭代回归 | 🟡 高 | Phase 1 开始补充核心测试 |
| 新闻 API 免费额度耗尽 | 🟡 中 | 监控 API 调用量 + 优化抓取频率 |

### 8.3 下一步行动建议

1. **立即**：修复假数据柱状图（0.5 天可完成）
2. **本周**：添加基础认证保护 + Rate Limiting
3. **下周**：统一中英文文案 + 添加操作反馈（Toast）
4. **两周内**：情绪趋势图 + 搜索功能
5. **一个月内**：完成 Phase 2 全部功能，产品进入可对外展示状态
